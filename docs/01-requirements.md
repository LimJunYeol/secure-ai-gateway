# Secure AI Gateway - 요구사항 명세서 v0.1

## 1. 프로젝트 한 줄 설명
기업 내부 문서·시스템을 LLM과 안전하게 연결하는 엔터프라이즈 AI Gateway.
RAG 검색과 MCP 도구 호출을 단일 게이트웨이가 통제하여
권한 / 감사 / 출처 / 민감정보 마스킹을 보장한다.

## 2. 해결하려는 문제
- 기업이 LLM/AI 에이전트를 도입할 때 다음을 통제할 수 없으면 도입 자체가 막힘:
  - 누가 어떤 데이터에 접근 가능한가
  - 답변의 출처가 무엇인가
  - 민감정보가 외부 LLM API로 유출되지 않는가
  - 사용자가 무엇을 묻고 무엇을 받았는지 추적 가능한가

## 3. 핵심 사용자
- 관리자 (Admin): 회사 등록, 사용자/그룹/권한 설정, 문서 업로드 정책 관리
- 일반 사용자 (Member): 채팅 인터페이스로 질문, RAG 답변과 MCP 도구 호출 결과 수신
- 감사자 (Auditor): 감사 로그 조회만 가능

## 4. MVP 기능 (4~6주 v1)
### F1. 인증 / 멀티테넌시
- 테넌트(회사) 가입 + 도메인 발급
- 사용자 가입 / 로그인 (JWT)
- 역할: Admin / Member (2개만)

### F2. 문서 인덱싱 (RAG)
- PDF, DOCX, Markdown 업로드
- 자동 청킹 + 임베딩 + 벡터 DB 저장
- 문서별 ACL: 어떤 그룹이 볼 수 있나
- 메타데이터: 업로더, 부서, 업로드 시간, 버전

### F3. RAG 질의
- 채팅 UI (간단)
- 검색 시점 권한 평가 (사용자 그룹 ∩ 문서 ACL)
- LLM 답변 + Citation (어느 청크가 사용됐는지)
- 답변 송출 전 민감정보 마스킹 (이메일, 전화번호, 주민번호)

### F4. MCP 도구 연동 (차별화 포인트)
- MCP Server 1개 이상 연결 (GitHub 권장)
- 같은 게이트웨이가 권한/감사/Rate Limit 적용
- End-to-end trace: 질문 → RAG → MCP → 답변 한 줄 추적

### F5. 감사 로그
- 모든 질문/답변/도구 호출 기록
- 관리자 대시보드에서 조회 (필터: 사용자, 기간, 도구)

### F6. Rate Limit + 비용 추적
- 사용자별 일/월 LLM 호출 한도
- 누적 토큰 사용량 표시

## 5. v1에서 제외 (의도적)
- 실시간 권한 동기화 (사용자 권한 변경 시 즉시 캐시 무효화)
- 도구 호출 샌드박싱 (네트워크 격리)
- 한국어 PII 고급 탐지 (정규식 수준만 v1)
- 멀티 LLM 라우팅 (단일 모델로 시작)
- 모바일 앱

## 6. 비기능 요구사항
- 응답 시간: RAG p95 < 5초 (LLM 응답 포함)
- 가용성: 단일 노드 99% (토이라서)
- 보안: 민감정보 외부 LLM API로 유출 0건 (마스킹 검증)
- 관측성: 모든 요청 trace_id 부여, 로그 수집

## 7. 성공 기준
- 데모 영상: "회사 가입 → 문서 업로드 → 다른 부서 사용자가 그 문서 접근 시도 → 거부 → 본인 부서 사용자가 질문 → 답변 + Citation + 마스킹 → 감사 로그 확인 → MCP 도구 호출"
- 부하 테스트: 동시 사용자 50명, p95 < 5초
- GitHub 레포: README + 아키텍처 다이어그램 + ADR 5개 이상

## 8. 의도적으로 안 다루는 부분 (면접 대응)
- 결제/구독: PG 통합은 토이 범위 밖
- 이메일 발송: 본질 아님
- SSO/OAuth: JWT만 (Keycloak은 v2)
